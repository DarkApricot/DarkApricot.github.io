Winter Wonder Ride!

This is project Winter Wonder Ride!

For this project we had 4-5 weeks with 2 developers (me being one of them) and 3 artists. We had 3 sprints total and managed to get a lot more done in the last sprint than expected, so the difference is quite hilarious.
In this project I made the menu's / UI, obstacles and general finetuning.

(For more in depth credits, go to the end of the 3rd video at the top.)

Sprint 1! - Alpha

This sprint was quite a mess, but we managed to get a good start. The beginning of the menu was there + general layout. Max, the other developer of this project, had started the movement of the car and we already had a checkpoint system.

The car movement is still very wonky and was practically impossible to drive, and everyone hit every obstacle possible as if they were drawn to it. ðŸ˜‚ The art also had a good base, and the blockout of the map was practically finished at this point.

Sprint 2! - Alpha

The 2nd sprint was even more of a mess, we did not manage to get a lot done and were quite behind on schedule. The car is more drivable, but it was still very hard to do so. It was bouncy and slid all over the map.

I had made icicle obstacles that dropped once you were close enough, which made everyone be stuck in the cave for a few minutes. The artists were already working on texturing of models and the menu had a few small updates here and there.

Sprint 3! - Beta

The third sprint was.. Insane. The artists had remade the entire map, the car movement was completely different with a handmade camera (Thank you, Max). The main and gameplay menus are completely finished with a scoreboard and checks so you can see how you did compared to the other saved files.

We added more obstacles: snowballs rolling down in the cave, snowpiles you have to avoid and the spikes falling were reworked. The map was now massive and with the finished power-ups much more fun to play!

Personal Challenges!

One of my biggest challenges (and eventually successes) of this project was managing multiple slots. I used PlayerPrefs for this, as I was unfamiliar with .Json files at the time. ðŸ˜­

I saved strings with specific information to numbers associated with each file. For example, names were saved in a string with "nameFile" + fileNumber, cars with "whichCar" + "file" (nameFile + fileNumber), and so on.

This type of saving was particularly tricky when I created the TOP10, as I had no way to associate file names with the now sorted top10. To address this, I created a scriptable object for each file. I spawned them with the correct visuals and named the objects by their deltaTime float value so I could sort the objects based on their names. I then displayed them in the UI according to the sorted array.
